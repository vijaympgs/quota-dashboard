# Quota Tracker — AI Coding Tools

Personal quota dashboard for AI coding tools across multiple Google accounts.

## Tools Tracked
| Tool | Fetch Method |
|---|---|
| Antigravity | `antigravity-usage` CLI |
| Claude Code | Anthropic Admin API |
| Kiro | Reference only |

## Setup

### 1. Install dependencies
```
pip install -r requirements.txt
```

### 2. Configure credentials
Copy `.env` and fill in your values:
```
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-secret
SESSION_SECRET=any-random-string
ANTHROPIC_ADMIN_KEY=sk-ant-admin-... # optional, for Claude Code
```

### 3. Google Cloud Console setup
1. Go to console.cloud.google.com
2. APIs & Services → Credentials → Create OAuth 2.0 Client ID
3. Application type: Web application
4. Authorized redirect URIs: `http://127.0.0.1:8000/auth/callback`
5. Copy Client ID and Secret to `.env`

### 4. Run
**Windows (double-click):**
```
start.bat
```

**Manual:**
```
python app.py
```

Then open: http://127.0.0.1:8000

## Usage
1. Click **+ Add Google account** in the sidebar
2. Sign in with Google → account appears in sidebar
3. Click **↻ Refresh** to fetch live quota data
4. Click any account to view its model usage cards
5. Click ✕ on an account to remove it from session

## File Structure
```
quota-dashboard/
  app.py              ← FastAPI server
  fetch_quota.py      ← Quota fetch logic
  static/
    index.html        ← Dashboard UI
  quota_data.json     ← Generated by Refresh
  .env                ← Your credentials (never commit this)
  requirements.txt
  start.bat
  README.md
```

## Notes
- `quota_data.json` is written locally and optionally pushed to GitHub Pages
- Credentials are stored in server session only — never sent to frontend
- Google token is used for session identity only — not for API calls
